# How to Create a Bug Report

_Generated on 2025-12-07_

Creating a bug report in the Bug Tracker is simple and straightforward. Follow these steps to document issues effectively.

## Steps to Create a Bug Report

1. **Navigate to the Dashboard**
   - Log in to your account
   - You'll land on the dashboard automatically

2. **Click "Create New Bug" Button**
   - Located in the top-right corner of the dashboard
   - The button is green and clearly visible

3. **Fill in the Required Fields**

   **Title** (Required)
   - Write a clear, concise summary of the bug
   - Example: "Login button not responding on Chrome"
   - Keep it under 100 characters

   **Description** (Required)
   - Provide detailed information about the bug
   - Include steps to reproduce the issue
   - Add any error messages you see
   - Mention browser, OS, or environment details

   **Severity** (Required)
   - **Critical**: System crash, data loss, security issue
   - **High**: Major feature broken, no workaround
   - **Medium**: Feature partially broken, workaround exists
   - **Low**: Minor issue, cosmetic problem

   **Priority** (Optional)
   - How urgent is the fix?
   - Low, Medium, High, Critical

   **Status** (Optional)
   - Usually starts as "Open"
   - Can be: Open, In Progress, Fixed, Closed, Reopened, Won't Fix

   **Assigned To** (Optional)
   - Select a team member to work on the bug
   - Leave blank if unsure

4. **Click "Submit" Button**
   - Review your information
   - Click the submit button at the bottom
   - You'll be redirected to the dashboard

5. **Verify Your Bug Appears**
   - Your new bug should appear in the dashboard table
   - It will have a unique ID number

## Tips for Writing Good Bug Reports

✅ **Be Specific**: Use clear, descriptive language
✅ **Include Steps**: List exactly how to reproduce the bug
✅ **Add Screenshots**: Visual evidence helps (if applicable)
✅ **One Bug Per Report**: Don't combine multiple issues
✅ **Check for Duplicates**: Search existing bugs first

## Common Issues & Fixes

**Issue**: Can't click the Submit button
- **Fix**: Make sure all required fields (Title, Description, Severity) are filled in

**Issue**: Don't see my bug after creating it
- **Fix**: Refresh the dashboard page or check if any filters are applied

**Issue**: Getting an error when submitting
- **Fix**: Check your internet connection and try again. If it persists, contact support.

## What Happens Next?

After creating a bug report:
1. It appears in the dashboard for all users to see
2. A manager can assign it to a team member
3. The assigned person can update the status as they work on it
4. You'll be listed as the reporter
5. You can edit or delete your bug report anytime
