# Getting Started with Bug Tracker

_Generated on 2025-12-07_

Welcome to the Bug Tracker! This guide will help you get up and running quickly.

## What is Bug Tracker?

Bug Tracker is a simple, web-based application for managing software bugs and issues. It helps teams:
- Document bugs clearly
- Track bug status from creation to resolution
- Assign bugs to team members
- Filter and search through bugs
- Maintain a history of all issues

## Quick Start (5 Minutes)

### Step 1: Log In

1. Open the Bug Tracker URL in your browser
2. You'll see the login page
3. Enter your credentials:
   - **Email**: Your registered email
   - **Password**: Your password

**Test Accounts:**
- Reporter: `reporter@example.com` / `password123`
- Manager: `manager@example.com` / `password123`

### Step 2: Explore the Dashboard

After logging in, you'll see the **Dashboard**:
- **Bug List Table**: All bugs in the system
- **Filter Options**: Dropdown to filter by status
- **Create Bug Button**: Green button in top-right
- **Navigation**: Links to Help Articles and logout

### Step 3: Create Your First Bug

1. Click the green **"+ Create New Bug"** button
2. Fill in the form:
   - **Title**: Brief description (e.g., "Login page not loading")
   - **Description**: Detailed info with steps to reproduce
   - **Severity**: Choose Critical, High, Medium, or Low
   - **Status**: Usually "Open" for new bugs
3. Click **"Submit"**
4. Your bug appears in the dashboard!

### Step 4: Manage Bugs

**View Bugs:**
- All bugs are visible in the dashboard table
- See ID, Title, Severity, Status, Reporter, Assigned To

**Edit a Bug:**
- Click the "Edit" button next to any bug you created
- Update fields as needed
- Click "Update Bug" to save

**Delete a Bug:**
- Click the "Delete" button next to bugs you created
- Confirm the deletion
- (Managers can delete any bug)

**Filter Bugs:**
- Use the dropdown above the table
- Select a status: Open, In Progress, Closed, etc.
- View only bugs with that status

## Understanding Your Role

Check the badge next to your email in the top-right:

**Reporter:**
- Create bugs
- Edit and delete your own bugs
- View all bugs
- Cannot edit/delete other people's bugs

**Manager:**
- Everything a Reporter can do
- Plus: Edit and delete ANY bug
- Generate reports

## Key Features to Explore

### Bug Statuses
- **Open**: New, not assigned yet
- **In Progress**: Being worked on
- **Fixed**: Solution implemented
- **Closed**: Verified and done
- **Reopened**: Previously closed but issue returned
- **Won't Fix**: Decision made not to fix

### Severity Levels
- **Critical**: System crash, data loss, security issue
- **High**: Major feature broken
- **Medium**: Feature partially works
- **Low**: Minor issue or cosmetic

### Priority
Set how urgent the fix is (separate from severity)

## Common Workflows

### Reporting a Bug
1. Notice an issue
2. Click "Create New Bug"
3. Fill in details thoroughly
4. Submit
5. Bug appears in dashboard

### Fixing a Bug
1. Manager assigns bug to you
2. You see it in the dashboard
3. Work on the fix
4. Update status to "In Progress"
5. When done, change to "Fixed"
6. Manager verifies and marks "Closed"

### Triaging Bugs (Managers)
1. Review new bugs daily
2. Assign to team members
3. Set priorities
4. Update statuses
5. Close verified fixes

### Troubleshooting

**Can't log in?**
- Check your email and password
- Make sure caps lock is off
- Contact your administrator

**Don't see a bug you created?**
- Check if filters are applied
- Refresh the page
- Make sure you're logged in

**Can't edit or delete a bug?**
- Check if you created it (Reporters only edit own bugs)
- Verify you have the right permissions
- Contact a manager for help
